# Hi, I'm Siddhant 👋

**I love solving real-world problems with code.**

I'm a software engineer who gets excited when a messy, ambiguous, real-world problem can be turned into something useful through clean engineering. Whether it's an AI-native workflow that automates tedious decision-making, a full-stack tool that saves people hours of work, or a model that finds patterns humans would miss — what hooks me is the gap between *"this is annoying / broken / slow"* and *"this just works now."*

Based in India, graduating in 2026, and constantly building.

---

### 🧭 How I work

- **Problems first, tools second.** I pick the stack that fits the problem, not the other way around.
- **Ship to learn.** I'd rather build a rough working version and improve it than read about it for a month.
- **Use AI to go faster, not skip the thinking.** I use coding assistants daily, but I don't paste code I can't explain line-by-line.
- **Real systems beat clever demos.** Validation, error handling, logging, and the boring 80% — that's where engineering actually lives.

---

I've been going deep on **AI-native engineering** — building applications where LLMs aren't just a chatbot bolt-on, but a real component in a workflow. RAG pipelines, agentic systems, structured outputs, evaluation, guardrails. The kind of work where you have to think about what happens when the model is wrong, not just when it's right.

Alongside that, I'm sharpening fundamentals: data structures, system design, and the production patterns that turn a working prototype into something a business can actually rely on.
